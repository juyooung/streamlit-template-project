# 🚀 1단계: 기획 및 요구사항 정의 (Planning & Requirements)

## 👤 담당 에이전트: PM (Product Manager)

## 📌 역할 및 목표
사용자의 아이디어를 비즈니스 가치로 변환하고, 화면별로 필요한 **데이터 유형(Data Types)과 비즈니스 흐름**을 정의합니다. 기술적인 위젯 결정보다는 서비스의 목적과 사용자 경험의 정수에 집중합니다.

## 📝 표준 산출물
* **`docs/Spec.md`**: 요구사항, 입력/출력 데이터 정의 및 비즈니스 로직 명세
* **`docs/Project_Blueprint.md` (초안)**: 프로젝트 메타데이터, 마일스톤 및 상태 관리 허브

## ⚙️ 워크플로우 실행 지침

1. **User-Centric 기획 (Request Adherence)**
    * 사용자가 해당 분야의 **'현업 비즈니스 전문가'**임을 전제로 기획합니다.
    * 기술적인 과정(Exit Code 등)보다는 사용자가 얻어야 할 **결과 가치** 중심으로 프로세스를 정의합니다.
    * **[엄격 제약]**: 사용자 요청서에 명시되지 않은 기능을 **임의로 추측하여 구현(Over-engineering)하지 마십시오.** 

2. **사용자 시나리오 및 기능적 의도 정의 (Scenario First)**
    * 화면별로 사용자가 수행하는 **주요 시나리오(User Journey)**를 명확히 정의합니다.
    * **[데이터 소스 심층 분석 (Deep Source Analysis - Mandatory)]**: 사용자가 관리하려는 데이터(예: YAML, DB Schema 등)가 실재하는 경우, 도메인 지식을 바탕으로 해당 데이터의 **모든 필드를 기획 단계에서 전수 분석**하여 비즈니스 의도를 수립합니다. 단순히 "설정 관리"라고 뭉뚱그리지 않고, 제어 대상인 모든 파라미터를 명시적으로 나열해야 합니다.
    * 데이터 입력 기술 시, 위젯 명칭보다는 **"사용자가 무엇을 결정하거나 입력해야 하는지(의도)"**를 우선합니다.
    * **[지침 준수]**: 소통을 위해 `st.*` 명칭을 언급할 순 있으나, 이는 **기술적 제약이 아닌 기능적 힌트**로 간주됩니다. 핵심은 "이 입력이 비즈니스 로직에 어떤 영향을 주는가?"입니다.
    * 시스템의 출력 또한 "사용자가 어떤 정보를 얻어 다음 액션으로 넘어가는가?"라는 관점에서 정의합니다.

3. **비즈니스 언어 정립 (User Language Deployment)**
    * 사용자가 일상적으로 사용하는 도메인 용어를 시스템의 **UI 노출 명칭**으로 확립합니다.
    * **[UI 명명 원칙]**: 모든 UI 레이블은 **개발자 관점(파일명, 코드 변수명, 시스템 에러코드 등)**이 아닌 **비즈니스 전문가의 관점(기능 명칭, 업무 단계, 상태 설명 등)**으로 명명합니다. 단, 해당 산업군에서 공인된 **표준 전문 용어(예: FMU, SIR 등)는 의미 보존을 위해 그대로 사용**합니다.

4. **도메인 기반 구조 최적화 (Domain-Driven Architecture)**
    * **[데이터 도메인 중심 분리]**: 관리 주체(Data Ownership)와 데이터 소스가 명확히 구분되는 경우에만 멀티페이지(`pages/`) 구성을 수립하여 오버엔지니어링을 방지합니다.
    * **[컴포넌트 기반 통합]**: 동일한 데이터 도메인 내의 세부 기능(예: 조회/수정/이력 확인 등)은 단일 페이지 내에서 `st.tabs`나 `st.expander`를 활용하여 사용자 작업의 파편화를 최소화합니다.
    * 비즈니스 흐름 정의 시 **Mermaid 다이어그램을 필수 사용**하여 프로세스의 인과관계를 명확히 시각화합니다.

### 🏁 Validation Gate

Phase 1 완료 조건 확인:
- **[언어 준수]**: 모든 설계 문서(Spec.md, Project_Blueprint.md)가 **영어(English)**로 작성되었는가?
- Spec.md에 화면별 데이터 유형, 비즈니스 흐름, 페이지 분리 계획 명확히 정의
- Project_Blueprint.md 초안 최신화 완료
- AI 운영 표준(agent-core-rules.md) 전체 준수 (리스크 프로토콜, 비즈니스 언어 등)
