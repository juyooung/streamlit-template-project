# 🚀 4단계: 검증 및 품질 보증 (Verification & QA)

## 👤 담당 에이전트: QA

## 📌 역할 및 목표
구현물이 기획(`Spec`) 및 설계(`Architecture`)와 100% 일치하는지 검증합니다. **비즈니스 언어(Business Language)** 준수 여부와 데이터 영속성을 전수 조사하여 완성도를 높입니다.

## 📝 표준 산출물
* **`docs/QA_Report.md`**: 전수 조사 결과, 버그 수정 내역 및 최종 승인 결과
* **`docs/Project_Blueprint.md` (최종 완료)**: 모든 기능 구현 및 검증 완료 상태 반영

## ⚙️ 워크플로우 실행 지침

1. **3-Pass 정밀 검증 (Multi-Pass Testing)**
    * **Pass 1 (Total Integration)**: 반드시 서비스의 **메인 엔트리포인트(예: `app.py`)를 실행**하여, 모든 페이지 내비게이션(사이드바 등)과 모듈 임포트가 최종 사용자 환경에서 무결한지 E2E로 확인합니다. 개별 모듈 단위의 분절된 검증은 '통과' 근거로 인정하지 않습니다.
    * **Pass 2 (State Persistence)**: 데이터 입력/저장 후 페이지 이동이나 새로고침 시 상태 유실 여부를 전수 조사합니다.
    * **Pass 3 (Edge Case & Performance)**: 비정상 파일 입력, 로딩 중 버튼 중복 클릭 등에 대한 시스템 안정성을 확인합니다.

2. **비즈니스 도메인 언어 감사 (Business Language Audit)**
    * UI 상의 모든 메시지(성공 안내, 에러 바 등)에 기술적인 흔적(파일명, 로우 에러코드 등)이 남아 있는지 확인합니다.
    * 표준 전문 용어(SIR, FMU 등)는 유지하되, 모든 텍스트가 기획 단계에서 정의된 실무자 관점의 용어로 표현되는지 검증합니다.

3. **데이터 정합성 및 무결성 확인 (Data Integrity)**
    * 시스템을 통한 데이터 수정 후, 실제 저장소(DB, Config 등)의 데이터가 의도한 형식과 정합성을 유지하는지 확인합니다.
    * 어떠한 경우에도 사용자 데이터의 유실(Loss)이나 불필요한 변형이 발생하지 않음을 검증합니다.

4. **최종 UX 및 성능 패스**
    * 페이지 로딩 속도와 위젯 반응성이 쾌적한지 확인합니다.
    * 캐싱(`@st.cache`)이 의도대로 작동하여 불필요한 재연산이 없는지 점검합니다.

### 🏁 Validation Gate

Phase 4 완료 조건 확인:
- 3-Pass 검증(기능/데이터 무결성/Business Language) 결과가 모두 통과되었는가?
- 발견된 모든 버그가 수정되었거나 Project_Blueprint.md에 공식 기록되었는가?
- Project_Blueprint.md의 최종 마일스톤 완료 및 아카이브 준비 완료
